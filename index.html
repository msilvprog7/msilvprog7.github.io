Projects

  • mshuffle
    A shuffling algorithm for your music playlists. Unlike a typical shuffle that simply picks a random song each time, mshuffle considers a history queue to prevent songs from being selected within a threshold from the last play. Also, users can (enjoy) or (dislike) a song to factor in to whether or not songs on the same album, by the same artist, or by related artists will be more or less likely for future song selections.
    Check out Node.js JavaScript
  • Arduino Web Controller
    Appeared at Hack.UVA 2016
    Often when making interesting LED displays, it's tedious to re-write similar Arduino code each time. Instead, Daniel Winker and I came up with a solution similar to Cloud and Amazon's Internet of Things with a web service for defining board layouts with RGB LEDs and controlling the board from the web. The Arduino code simply defines the RGB LED groups, then a client scripts registers it with the web service so then you can collaboratively control the board. This project also makes it easy to write scripts that change the board based on data from the web, such as the Twitter Stream feature that can change the LEDs based on three user-defined topics of interest.
    Check out Python (Flask) Arduino JavaScript
  • RTF's Trail Management Software (Capstone)
    A year-long capstone project at the University of Virginia
    For my fourth-year capstone project, I was on a team of 7 that developed a website for the Rivanna Trails Foundation. Specifically, they needed a way to start collecting right-of-way property agreements for properties crossed by the 20+ miles of trail in Charlottesville, VA, and Albemarle County. Our system ingests local property records as well as the KML file for the Rivanna trail. We then intersect the trail with the properties to break the trail into one segment per property. The trail management system the entire trail and allows the board members to change the trail and reference property agreements.
    Not available Python (Django) Google Maps JS API JavaScript HTML5
  • JS Game Engine / Master Biomancer
    A Game Design course project
    In my fourth-year at the University of Virginia, I took a Game Design course where I designed a game engine as well as a game using the engine with 2 other students. The game engine is based on simple principles like display hierarchies with axis-aligned bounding boxes for hit boxes. Additionally, we made a level editor to simplify level creation. We then made Master Biomancer using the engine. The game is a single-player, shooter game where the main character, the Biomancer, synthesizes animals from his gun to help fight off the enemies in his lab. Using the abilities of synthesized wolves, spiders, turtles, and penguins, the player must skillfully navigate through each of the levels to save his science from the enemies invading his lab.
